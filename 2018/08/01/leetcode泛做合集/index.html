leetcode泛做合集

[34. Find First and Last Position of Element in Sorted Array]
二分最小的$l$,使得$nums[l] \ge target$.二分最大的$r$,使得$nums[r] \le target$.

[35. Search Insert Position] :
二分最小的$l$,使得$nums[l] \ge target$.
注意一个细节:如果$l$在vector的末尾,需要特殊判断.

[36. Valid Sudoku]
解法一:分 行,列,方格 讨论,判断是否出现重复数字.时间复杂度$O(3*9*(9+9))$.
解法二:用维护$3*9$个Hash表,只需要扫描一次棋盘即可.时间复杂度为$O(9*9)$.

[37. Sudoku Solver]
直接暴力Dfs即可,可以用T36的做法二来记录某些数字是否用过.

[38. Count and Say]
递推模拟,将字符串分解成连续的若干段,统计每段对应的数字和个数$t$,其中$t$可能不止是个位数.

[39. Combination Sum]
简单的Dfs

[40. Combination Sum II]
同39,注意在解的出口加判重

[41. First Missing Positive]
思维好题,有两种解法.
解法一:考虑没有额外空间限制的问题.引入辅助数组$A[]$,初始化全为$1$.在扫描数组的时候,将$nums[i]$放入$A[nums[i]-1]$下,最后扫描$A[]$,遇到第一个为$-1$的位置跳出即可.现在考虑原问题,将$num[]$和$A[]$合并,仍然扫描$nums[]$,将$nums[i]$和$nums[nums[i]-1]$交换(注意可能会一直交换下去).
解法二:同样考虑没有限制的问题.引入辅助数组$A[]$,初始化全为$1$.不妨设$nums[i]>0$(否则将$le 0$的部分全部放到后面).扫描数组的时候,将$A[nums[i]-1]=-1$.最后扫描$A[]$,碰到第一个为$1$的位置跳出即可.现在考虑原问题,将$nums[]$和$A[]$合并，仍然扫描$nums[]$,当扫描到$nums[i]$时,令$nums[abs(nums[i])-1]$变为负数即可.最后只需要扫描$nums[]$,在第一个不是负数的地方跳出即可.
